Autonomous Robotic Arm Simulation

A C++ based simulation applying inverse kinematics and control theory, delivered ahead of schedule using Agile methodologies.

Role: Lead Developer, Project Manager
Tech Stack: C++, Python, ROS
Duration: 12 Weeks
Outcome: Delivered - 3 Days Early

Problem Statement

The goal was to simulate a 6-DOF robotic arm to validate a new control algorithm within a 12-week semester, with no budget for physical hardware. This required a robust physics simulation and an accurate kinematic model.

Code Spotlight: Inverse Kinematics

// Jacobian-based solver for inverse kinematics
void RobotArm::calculateIK(Vector3 target) {
    const float alpha = 0.1;
    for (int i = 0; i < MAX_ITERATIONS; ++i) {
        Vector3 currentPos = getEndEffectorPosition();
        Vector3 error = target - currentPos;
        if (error.length() < IK_THRESHOLD) {
            return; // Success
        }
        Matrix3x3 jacobian = calculateJacobian();
        Matrix3x3 pseudoInverse = jacobian.pseudoInverse();
        Vector3 deltaTheta = pseudoInverse * error;
        applyJointAngles(m_jointAngles + deltaTheta * alpha);
    }
}

Personal Reflection

This project was a phenomenal learning experience. The biggest takeaway was the trade-off between algorithmic complexity and real-time performance. While a more complex solver could yield slightly higher accuracy, the Jacobian-based method provided the best balance for this application. In the future, I would explore implementing a Kalman filter to further smooth sensor data before it enters the control loop.
